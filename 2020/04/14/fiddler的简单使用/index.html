fiddler的简单使用

今天介绍一个抓包工具，和charles功能类似，界面要差一点，但是呢，这个软件功能要强大的多，最重要的是免费，哈哈，更多功能期待你的发掘

提问：fiddler能给测试工作带来什么帮助？
1、帮助测试同学确定问题出在哪个端，是否是服务端的问题。
2、需要一些你想要显示的内容，但是通过改服务端数据等方式很麻烦，浪费时间，你可以通过抓包工具篡改下数据，这样更加节省时间。
3、查看接口的请求参数或者返回参数。
4、查看接口的响应时间等，分析性能。
5、模拟弱网环境。

接下来进入正题

一、首先，我们需要安装一个charles软件。当然可以去官网或者其他网站，下载汉化版什么的，这里就不多赘述了。
fiddler的官方网站（https://www.telerik.com/fiddler）

二、查看本机的ip
方法1、cmd输入ipconfig，查看ipv4后就是本机的IP地址

三、配置软件。

1、进入Tools –> Fiddler Options –> HTTPS
2、全勾上就行

意思呢，就是破解你的https的流量官方一点的话就是：
Fiddler可以通过伪造CA证书来欺骗浏览器和服务器。Fiddler是个很会装逼的好东西，大概原理就是在浏览器面前Fiddler伪装成一个HTTPS服务器，而在真正的HTTPS服务器面前Fiddler又装成浏览器，从而实现解密HTTPS数据包的目的。

此时你就可以监听由当前电脑的所有请求啦！

当然，需要测试客户端的话这还是不够的，我们需要为我们的设备设置代理。

一、首先，我们需要打开Fidder，点击菜单栏中的 [Tools] –> [Fiddler Options]->[Connections]，设置你的端口号，一般是8888，当然你也可以随便填个别的，记住就好。勾选 Allow remote computers to connect， 点击OK

二、在移动端的浏览器中输入：ip地址+:端口号（例如：192.168.1.1:8888） ,下载证书并安装在你的手机中。
ps：ios需要在设置-通用-关于手机中信任这个证书。

三、在你的WiFi后配置手动代理
服务器填你的电脑ip ，端口填默认的8888，当然需要其他的端口可以在charles里修改。这俩就不多描述了

四、操作你的手机，譬如打开玩吧的app，此时你就看到抓取的所有接口啦！

现在你已经能看到所有的请求啦，但是你如果想用其他的功能，请继续往下看！

1、查看一些接口的基本信息,请求类型协议响应时间等等

2、查看接口的请求参数和返回参数

Fiddler 内置命令与断点
Fiddler还有一个藏的很深的命令框，FIddler断点功能就是将请求截获下来，但是不发送，这个时候你可以干很多事情，比如说，把包改了，再发送给服务器君。还有balabala一大堆的事情可以做。这里就不多赘述了。因为我现在也不太会用，哈哈

命令 对应请求项 说明 例子
? All 问号后边跟一个字符串，可以匹配出包含这个字符串的请求 ?google
> Body 大于号后面跟一个数字，可以匹配出请求大小，大于这个数字请求 >1000
< Body 小于号跟大于号相反，匹配出请求大小，小于这个数字的请求 <100
= Result 等于号后面跟数字，可以匹配HTTP返回码 =200
@ Host @后面跟Host，可以匹配域名 @www.baidu.com
select Content-Type select 后面跟响应类型，可以匹配到相关的类型 select image
cls All 清空当前所有请求 cls
dump All 将所有请求打包成saz压缩包，保存到“我的文档\Fiddler2\Captures”目录下 dump
start All 开始监听请求 start
stop All 停止监听请求 stop
命令 对应请求项 说明 例子
bpafter All bpafter后边跟一个字符串，表示中断所有包含该字符串的请求 bpafter baidu（输入bpafter解除断点）
bpu All 跟bpafter差不多，只不过这个是收到请求了，中断响应 bpu baidu（输入bpu解除断点）
bps Result 后面跟状态吗，表示中断所有是这个状态码的请求 bps 200（输入bps解除断点）
bpv / bpm HTTP方法 只中断HTTP方法的命令，HTTP方法如POST、GET bpv get（输入bpv解除断点）
g / go All 放行所有中断下来的请求 g

断点可以直接点击Fiddler下图的图标位置，就可以设置全部请求的断点，断点的命令可以精确设置需要截获那些请求。如下示例：

当然你也可以选择一个接口右键，选择重做-重发并编辑，可以快速的进行断点。
1、直接点击界面内的值或者在choose response中选择本地文件

当然你还可以直接设置本地转发
一、在右侧工具栏中选择自动转发，上面的对勾都勾上在里面填写你的信息并启用，保存

然后再次请求下这个接口，之后的返回值就是你自己的本地文件修改的内容啦！很简单吧

关于模拟弱网环境
打开Fiddler，Rules->Performance->勾选 Simulate Modem Speeds，勾选之后访问网站会发现网络慢了很多

修改弱网模拟的速度

Rules—>Cutomize Rules打开CustomRules.js 文档

在文件中搜索关键字，m_SimulateModem，如图所示

让我们来看看一下这几行代码怎么设置：

首先来判断m_SimulateModem是否为true，也就是是否设置了弱网模式。

如果为弱网模式。则分析代码

oSession[“request-trickle-delay”] = “300”; 意思就是：上传1KB需要300ms，
同样的方法，也可以限制上传的速度，调整oSession[“response-trickle-delay”]即可。
