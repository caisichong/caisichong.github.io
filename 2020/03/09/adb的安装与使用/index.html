Windows下adb的配置及使用

ADB环境配置：

方法一：直接安装adb

   下载文件：ADB.zip，解压到你喜欢放在的一个地方

   在电脑的高级设置里面把环境变量Path改下就OK，在Path后面追放你的adb绝对路径（如上图:）注意前面以；隔开
   配置环境：步骤：右键此电脑-->属性--->高级系统设置---->环境变量----->系统变量-------->Path


再次确认ADB安装
配置好之后检查一下，Windows开始键+R打开命令提示符----->输入cmd确认---->输入adb

方法二：安装Android studio

三：设备开启开发者选项：

 包括 Android 设备处于正常开机状态，USB 连接线和各种接口完好。
 Android 设备的开发者选项和 USB 调试模式已开启。
可以到「设置」-「开发者选项」-「Android 调试」查看。
如果在设置里找不到开发者选项，那需要通过一个彩蛋来让它显示出来：在「设置」-「关于手机」连续点击「版本号」7 次

常用命令：

1.显示当前链接的设备：

 adb devices

常见异常输出：

没有设备/模拟器连接成功。
 List of devices attached 设备/模拟器未连接到 adb 或无响应。
 List of devices attached cf264b8f offline USB 连接
如果能看到，xxxxxx device 说明连接成功。

2.清除手机已有log：

adb logcat -c

3.抓取安卓手机日志：

adb logcat -v time >e:/log20190101.txt

清除应用数据及缓存：

adb shell pm clear
4.重启手机：

adb reboot

 格式

 reboot [－n] [－w] [－d] [－f] [－i]

 主要参数 ：

 －n: 在重开机前不做将记忆体资料写回硬盘的动作。

 －w: 并不会真的重开机，只是把记录写到/var/log/wtmp文件里。

 －d: 不把记录写到/var/log/wtmp文件里（－n这个参数包含了－d）。

 －i: 在重开机之前先把所有与网络相关的装置停止。

5.打印当前时间：

adb shell date

6.查cpu信息：

adb shell cat /proc/cpuinfo

CPU数据获取：
adb shell dumpsys cpuinfo | grep packagename
top命令
adb shell top -m 10 -s cpu #查看占用cpu最高的前10个程序（-t 显示进程名称，-s 按指定行排序，-n 在退出前刷新几次，-d 刷新间隔，-m 显示最大数量）
adb shell top | grep PackageName > /address/cpu.txt

7.获取分辨率：

adb shell wm size

8.查内存：

adb shell dumpsys meminfo

adb shell cat /proc/meminfo // 查看内存信息

9.查看所启动的activity：

adb logcat ActivityManager:I *:s

查看前台activity ：adb shell dumpsys activity activities | grep mFocusedActivity

10.启动指定Activity：

adb shell am start -n 包名/包名＋类名（-n 类名,-a action,-d date,-m MIME-TYPE,-c category,-e 扩展数据,等）。

11.查询包的路径：

adb shell pm path 包名

12.push包：

adb pull 包的路径 e:\

13.安装应用程序：

 直接安装：

 adb install 应用名称.apk

 覆盖安装：

 adb install -r 应用程序.apk   （若出现错误，先看下：设置->安全->未知来源（是否已允许安装来自未知来源的应用）。）

安装到sd卡：

 adb install -s apk.apk  

 降级安装：

 adb install -d apk.apk  

 允许所有权限：

 adb install -g apk.apk  

 安装到指定设备中 ：

 adb -s  设备名称  install 应用名称.apk

14：缷载apk包：

方法一（需要root权限）：

adb shell

 cd data/app

 rm apk包

 exit

 方法二： adb uninstall apk包的主包名

 方法三： adb install -r apk包

  1. 删除系统应用：

    adb remount （重新挂载系统分区，使系统分区重新可写）。

adb shell

cd system/app

rm *.apk

16.获取管理员权限：

adb root

adb shell   若为#号则为root成功若为 $，则无root权限

 adb  unroot   移除root权限

17.查看bug报告：

adb bugreport

18.记录无线通讯日志：

adb shell

logcat -b radio

19.获取设备的ID和序列号：

adb get-product

adb get-serialno

20.访问数据库SQLite3：

adb shell

sqlite3

查看应用列表：

 adb shell pm list packages [-f] [-d] [-e] [-s] [-3] [-i] [-u] [–user USER_ID] [FILTER]
             -f 显示应用关联的 apk 文件

            -d 只显示 disabled 的应用

            -e 只显示 enabled 的应用

            -s 只显示系统应用

            -3只显示第三方应用

             -i 显示应用的 installer

            -u 包含已卸载应用                        

            例如：

adb shell pm list packages -s 第三方应用
adb shell pm list packages -3 包名包含某字符串的应用
            比如要查看包名包含字符串 mazhuang 的应用列表，命令：

adb shell pm list packages mazhuang 当然也可以使用 grep 来过滤：
adb shell pm list packages | grep mazhuang 安装 APK

启动 adb server 命令：

adb start-server （一般无需手动执行此命令，在运行 adb 命令时若发现 adb server 没有启动会自动调起。）

停止 adb server 命令：

adb kill-server

查看adb版本：

adb version
